National Vulnerability Database (NVD) CVE Dashboard

Overview

A comprehensive full-stack application for fetching, storing, and displaying CVE (Common Vulnerabilities and Exposures) data from the National Vulnerability Database. This dashboard provides security professionals and developers with real-time access to vulnerability information with powerful filtering and search capabilities.

Technology Stack

Backend: Node.js, Express, MongoDB, Mongoose, Axios, Zod, node-cron

Frontend: Vanilla JavaScript, HTML5, CSS3

Data Source: NVD REST API v2.0

Available Filters

  • CVE ID: Search by specific CVE identifier
  • Year: Filter by publication year (YYYY)
  • Score Min: Minimum CVSS base score (0-10)
  • Score Version: CVSS v2 or v3
  • Modified Last Days: Recently updated CVEs
  • Keyword: Search in vulnerability descriptions

Data Processing

The system automatically fetches CVE data from the NVD API, normalizes the information, removes duplicates, and stores it in MongoDB. A scheduled job runs daily at 00:30 to fetch incremental updates, ensuring the database stays current with the latest vulnerability information.

API Endpoints

GET /api/cves - List CVEs with filtering and pagination

GET /api/cves/:id - Get detailed information for a specific CVE

GET /api/health - Check API health status
